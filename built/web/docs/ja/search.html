検索

投稿を検索することができます。

キーワードを半角スペースで区切ると、and検索になります。 例えば、「git コミット」と検索すると、「gitで編集したファイルの特定の行だけコミットする方法がわからない」などがマッチします。

キーワードの除外

キーワードの前に「-」(ハイフン)をプリフィクスすると、そのキーワードを含まない投稿に限定します。

例えば、「gitというキーワードを含むが、コミットというキーワードは含まない投稿」を検索したい場合、クエリは以下のようになります:

git -コミット

完全一致

テキストを「"""」で囲むと、そのテキストと完全に一致する投稿を検索します。

例えば、「"""にゃーん"""」と検索すると、「にゃーん」という投稿のみがヒットし、「にゃーん…」という投稿はヒットしません。

タグ

キーワードの前に「#」(シャープ)をプリフィクスすると、そのキーワードと一致するタグを持つ投稿に限定します。

オプション

オプションを使用して、より高度な検索を行えます。 オプションを指定するには、「オプション名:値」という形式でクエリに含めます。

利用可能なオプション一覧です:

名前説明
user指定されたユーザー名のユーザーの投稿に限定します。 「,」(カンマ)で区切って、複数ユーザーを指定することもできます。
例えば、user:himawari,sakurakoと検索すると「@himawariまたは@sakurakoの投稿」だけに限定します。 (つまりユーザーのホワイトリストです)
exclude_user指定されたユーザー名のユーザーの投稿を除外します。 「,」(カンマ)で区切って、複数ユーザーを指定することもできます。
例えば、exclude_user:akari,chinatsuと検索すると「@akariまたは@chinatsu以外の投稿」に限定します。 (つまりユーザーのブラックリストです)
followtrue ... フォローしているユーザーに限定。
false ... フォローしていないユーザーに限定。
null ... 特に限定しない(デフォルト)
mutemute_all ... ミュートしているユーザーの投稿とその投稿に対する返信やRepostを除外する(デフォルト)
mute_related ... ミュートしているユーザーの投稿に対する返信やRepostだけ除外する
mute_direct ... ミュートしているユーザーの投稿だけ除外する
disabled ... ミュートしているユーザーの投稿とその投稿に対する返信やRepostも含める
direct_only ... ミュートしているユーザーの投稿だけに限定
related_only ... ミュートしているユーザーの投稿に対する返信やRepostだけに限定
all_only ... ミュートしているユーザーの投稿とその投稿に対する返信やRepostに限定
replytrue ... 返信に限定。
false ... 返信でない投稿に限定。
null ... 特に限定しない(デフォルト)
reposttrue ... Repostに限定。
false ... Repostでない投稿に限定。
null ... 特に限定しない(デフォルト)
mediatrue ... メディアが添付されている投稿に限定。
false ... メディアが添付されていない投稿に限定。
null ... 特に限定しない(デフォルト)
polltrue ... 投票が添付されている投稿に限定。
false ... 投票が添付されていない投稿に限定。
null ... 特に限定しない(デフォルト)
until上限の日時。(YYYY-MM-DD)
since下限の日時。(YYYY-MM-DD)

例えば、「@syuiloの2017年11月1日から2017年12月31日までの『Misskey』というテキストを含む返信ではない投稿」を検索したい場合、クエリは以下のようになります:

user:syuilo since:2017-11-01 until:2017-12-31 reply:false Misskey
